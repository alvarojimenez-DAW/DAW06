DAW06

Proces de desenvolupament del projecte Kanban

Aplicacio web de gestio de tasques amb HTML, CSS, JavaScript i persistencia amb localStorage.

Repositori GitHub: https://github.com/alvarojimenez-DAW/DAW06

GitHub Pages: https://alvarojimenez-daw.github.io/DAW06/

Issue 1 - Inicialitzacio del projecte i estructura base

S'ha creat l'estructura minima del projecte amb index.html, css/estils.css, js/script.js, img/, docs/ i README.md. L'HTML utilitza etiquetes semantiques com header, main, aside, section i footer.

La pagina inclou un formulari per crear i editar tasques, controls de filtre i cerca, estadistiques globals i un tauler Kanban amb tres columnes: Per fer, En curs i Fet.

Issue 2 - Model de dades i persistencia amb localStorage

El model de tasca inclou id, titol, descripcio, prioritat, dataVenciment, estat i creatEl. Les funcions carregarTasques() i guardarTasques() gestionen la persistencia amb la clau tasquesKanban.

Quan s'inicia l'aplicacio, es recuperen les dades desades i es renderitza el tauler.

Issue 3 - CRUD complet de tasques i renderitzacio del Kanban

S'ha implementat la creacio de tasques amb validacio del titol obligatori, l'edicio amb formulari preomplert, l'eliminacio amb confirmacio i el canvi d'estat des de cada targeta.

La funcio renderTauler() centralitza la renderitzacio de columnes i targetes. Cada canvi actualitza immediatament el DOM i el localStorage.

Issue 4 - Filtres, cerca i estadistiques

Els filtres permeten seleccionar estat i prioritat, i el camp de cerca treballa sobre titol i descripcio sense distingir majuscules i minuscules. La funcio getTasquesFiltrades() separa la logica de filtratge de la interfície.

Les estadistiques globals mostren total de tasques, recompte per estat i percentatge de tasques completades.

Issue 5 - Responsive, Git flow i documentacio final

El CSS utilitza Grid i Flexbox. En pantalles petites el formulari, filtres i columnes s'apilen verticalment per mantenir la usabilitat. El README documenta la funcionalitat, l'us basic, l'estructura del projecte i els enllacos pendents de GitHub.

El projecte queda preparat per publicar-se a GitHub Pages des de la branca main. El README inclou els enllacos del repositori i de la pagina desplegada, a mes d'una captura principal de l'aplicacio.

Problemes i solucions

Per garantir compatibilitat amb GitHub Pages, s'han utilitzat rutes relatives i no s'ha configurat cap proces de compilacio. La persistencia es fa amb localStorage, que es suficient per a una practica de frontend sense backend.

Comprovacions finals

La pagina es pot obrir en local amb index.html. Les tasques es creen, s'editen, s'eliminen, es mouen entre estats i es mantenen despres de recarregar la pagina.
